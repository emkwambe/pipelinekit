# PipelineKit Glossary

## Pipeline

A reproducible workflow that moves, validates, transforms, diagnoses, and reports on data.

## Blueprint

A reusable implementation pattern for a trusted analytics workflow.

## Contract

A machine-readable statement of expectations about data structure, freshness, quality, and business rules.

## Diagnosis

An evidence-based explanation of a pipeline or analytics failure.

## Trust

The confidence that analytics outputs are correct, current, explainable, and fit for decision-making.

## State

Metadata about pipeline execution, validation, diagnosis, and alerts.

## Adapter

A provider-specific implementation behind a stable PipelineKit interface.

## Specification

An implementation-ready document defining one buildable system capability.
